# RAG Copilot

Single-topic RAG QA for a single document domain. The app supports local document upload, chunking, vector retrieval, grounded answers with citations, and an offline evaluation loop.

## What it does

- Upload documents from the browser.
- Rebuild the local index after upload or deletion.
- Ask questions with a grounded answer generated by DashScope/Qwen or a mock model.
- Return citations for each answer.
- Hide documents from the index without deleting the local file.
- Run a small offline evaluation set.

## Supported inputs

- Markdown: `.md`, `.markdown`
- Plain text: `.txt`
- FAQ CSV: `.csv`
- PDF: `.pdf`

PDF support is included in the default backend dependencies.

## Requirements

- Python 3.11+
- Node.js 18+ and npm
- A DashScope / Qwen API key if you want real model answers

## Quick start

### 1) Clone the repository

```powershell
git clone https://github.com/adoleguizi/rag-copilot.git
cd rag-copilot
```

### 2) Set up Python dependencies

Recommended with `uv`:

```powershell
uv sync
```

Fallback with `pip`:

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

### 3) Set up the frontend

```powershell
npm --prefix frontend install
```

### 4) Create `.env`

Create a `.env` file in the project root:

```env
DASHSCOPE_API_KEY=your_api_key_here
DASHSCOPE_MODEL=qwen3-omni-flash-2025-12-01
DASHSCOPE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
```

Optional:

```env
RAG_LLM_PROVIDER=dashscope
RAG_CORS_ORIGINS=http://127.0.0.1:5173,http://localhost:5173
```

Do not commit `.env`.

## How to use

### Upload documents

1. Open the frontend.
2. Select `.txt`, `.md`, `.csv`, or `.pdf` files.
3. Click `Upload`.

Uploaded files are stored locally under `data/raw_docs/`.

### Rebuild the index

Click `Rebuild Index` after uploading or removing documents.

The backend rebuilds `data/index/vector_index.json` from the current document set.

### Ask a question

Choose a model:

- `DashScope` for real answers
- `Mock` for offline testing

Then ask a question and read the answer plus citations.

### Remove a document from the index

Click `Remove from index`.

This is a soft delete:

- the local file is kept on disk
- the file is hidden from the document list
- the file is skipped during index rebuild

If you re-upload the same file name, it becomes visible again.

## Testing and evaluation

Run the Python test suite:

```powershell
.\.venv\Scripts\python.exe -B -m unittest discover -s tests -p "test_*.py"
```

Run a simple evaluation:

```powershell
.\.venv\Scripts\python.exe scripts\run_eval.py --llm dashscope --top-k 2 --show-details
```

Use `--llm mock` if you want to avoid model API calls.

## Important notes

- The app is a local web app, not a hosted SaaS.
- Other users need to clone the repo and run it locally.
- The frontend never receives your DashScope API key. The key stays in the backend `.env`.
- If you delete a file from the UI, the file is not removed from disk.
- PDF upload works without extra manual dependency steps because `pypdf` is included in the backend dependencies.

## Common commands

Build the index from the CLI:

```powershell
.\.venv\Scripts\python.exe scripts\build_index.py
```

Start the search-only debug CLI:

```powershell
.\.venv\Scripts\python.exe scripts\search_index.py "your question"
```

Ask from the CLI:

```powershell
.\.venv\Scripts\python.exe scripts\ask.py "your question" --llm dashscope
```
